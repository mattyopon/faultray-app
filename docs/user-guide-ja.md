# FaultRay SaaS ユーザーガイド

> 本番環境を壊さずに、システムの弱点を自動で見つけます。

---

## はじめに

FaultRayは、インフラの障害リスクをシミュレーションで可視化するSaaSです。このガイドでは、SaaSダッシュボードの主要機能と使い方を説明します。

CLI版のドキュメントは [GitHub](https://github.com/mattyopon/faultray) を参照してください。

## 2. ダッシュボード

ログイン後のメイン画面です。

### 表示内容
- **リスクスコア**: インフラ全体の健全性を0-100で表示
- **最近のシミュレーション結果**: 直近の実行履歴
- **アラート**: 検出されたリスクの一覧
- **クイックスタート**: 初回ユーザー向けのガイド

---

## 3. シミュレーション実行

### 3-1. テンプレートから実行

1. サイドバーの「Simulate」をクリック
2. テンプレート一覧から業種・構成を選択:
   - Web SaaS (3-tier)
   - E-commerce Platform
   - Healthcare EHR
   - Fintech Banking
   - Hybrid On-Prem Cloud
3. 「Run Simulation」をクリック
4. 結果が表示されるまで約1-2分

### 3-2. カスタム構成で実行

1. 「Simulate」→「Custom」タブ
2. YAML形式でインフラ構成を入力、またはファイルアップロード
3. 「Run Simulation」をクリック

### 3-3. Terraform連携

1. 「IaC」ページからTerraform stateファイルをアップロード
2. FaultRayが自動的にインフラ構成を解析
3. リスク診断結果を表示

---

## 4. 結果の見方

### リスクスコア

| スコア | 意味 | 推奨アクション |
|--------|------|---------------|
| 90-100 | 優秀 | 現状維持 |
| 70-89 | 良好 | 軽微な改善推奨 |
| 50-69 | 要注意 | 改善計画の策定 |
| 0-49 | 危険 | 即座の対応が必要 |

### レポート内容
- **影響範囲マップ**: どのコンポーネントが落ちると何に影響するか
- **推定損害額**: 障害1件あたりの推定損失額
- **改善提案**: 具体的な改善アクションリスト
- **DORA指標**: デプロイ頻度、リードタイム、MTTR、変更失敗率
